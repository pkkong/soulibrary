# 작업 메모 (공용)

## 사용 가이드 (Codex용)
- 새 작업 시작 전에 이 문서를 읽고 상태를 확인합니다.
- 새 작업/메모를 추가할 때는 사용자에게 “tasks.md에 반영할까요?”를 먼저 묻고, 동의 시 추가합니다.
- 완료한 작업은 “완료”로 옮기고 날짜/간단 메모를 남깁니다.
- 불필요해진 항목은 사용자 확인 후 삭제하거나 “보류/미룸” 섹션을 새로 만들어 이동합니다.
- 코드 작성 시 한글이 깨지지 않도록 항상 인코딩(UTF-8)과 문자열 처리 방식을 확인합니다.
- 추정/가정으로 구현하지 말고, 확인이 필요한 사항은 사용자에게 물어본 뒤 진행합니다.
- 파싱 규칙/데이터 해석은 제공된 HTML/샘플 기준으로만 반영하고, 부족한 정보는 사용자에게 확인합니다.

## 해야 할 일
- 검색 오류 재확인: 현재 MATCH 표현식(`"토큰* OR 토큰*"`)으로 여전히 오류 발생 → 단순 `query*`로 복구 여부 검토, 에러 로그 확보.
- 서울교육청 원격 권수: 소장형(sen_owned) 500/실패 로그 확인(콘솔에 찍힌 `[total_count api] sen_owned ...` 메시지).
- 서울교육청 상태 초기화: `/admin/reset/sen_owned`, `/admin/reset/sen_subs` 호출 후 idle 확인, 필요 시 재실행.
- `data/library.db` 재생성 후 `/admin`/검색 정상 동작 재확인(500/no-table 에러 재발 확인).
- 서울교육청(구독) 오디오 처리 결정: 원격 권수와 로컬 차이는 AUDIO 제외 때문. 옵션 결정 필요
  - A) AUDIO 포함 수집으로 변경해 숫자 맞추기
  - B) AUDIO는 계속 제외하되 대시보드에 “오디오 제외로 차이 발생” 표시/권수 비교 로직 조정
- 서울교육청 소장 원격 권수: totalCount 문자열 응답도 정수 변환하도록 수정함. “원격 권수 새로고침” 후 콘솔 로그(`[total_count api] sen_owned ...`) 확인 필요.
- 검색 FTS 재빌드: `books_fts`를 `tokenize='unicode61'`로 재생성해 한글 접두 검색 오류 수정됨. `data/library.db` 재빌드 완료(1,627,305권). 서버 재시작 후 검색 정상 동작 확인 필요.
- 검색 스토어 개선 검토: SQLite 빌드 시간/용량 문제 → 최적화 또는 Meilisearch/Typesense 등 전용 검색엔진 POC 비교.

## 진행 중
- 서울교육청 소장/구독: 중복 제거(dropping duplicates) 제거 완료. CSV에는 모든 행이 남음.
- 원격 권수 로직: 소장/구독 모두 실제 목록 호출 파라미터로 요청하도록 수정.
- 원격 권수 파서: 소장형 totalCount가 문자열일 때도 정수로 처리하도록 수정(로그 확인 필요).

- 상태 자동 초기화: running 1시간 이상 stale 시 idle로 초기화.

## 단계적 전환 계획 (현행 유지 → 신규 DB 병행)
- 1단계: 기존 검색 유지, 새 DB(books/holdings) 빌드 스크립트만 추가해 병행 생성.
- 2단계: 샘플/카운트 비교 스크립트로 새 DB 품질 검증(검색 결과/도서관 매핑).
- 3단계: 검색 API를 새 DB로 전환(롤백 가능하도록 스위치/플래그 추가).
- 4단계: 기존 단일 DB 빌드 경로 정리(문서/배포 플로우 업데이트).
