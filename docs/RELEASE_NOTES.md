# 릴리즈 노트

## 2026-05-19 - 내 서재 UX 정리

### 이번 버전에서 바뀐 점
- 사용자가 책을 따로 모아둘 수 있는 `내 서재` 화면을 추가했다.
- 하단 내비게이션에 `서재` 진입점을 추가해 검색 후 저장한 책으로 바로 돌아갈 수 있게 했다.
- 검색 결과 카드는 기존처럼 카드 전체를 누르면 상세로 이동하는 구조를 유지했다. 검색 카드 안에 별도 저장 버튼을 넣지 않아 목록의 단순함을 지켰다.
- 책을 내 서재에 담는 동작은 책갈피 아이콘으로 통일했다. 상세 화면 상단과 검색 결과 카드 우측에서 같은 방식으로 담을 수 있다.
- 검색 결과의 책갈피 버튼은 커버를 가리지 않도록 카드 우측 보조 액션으로 옮겼다.
- 검색 결과 카드의 상세 이동 chevron을 제거했다. 카드 전체는 상세 이동, 우측 책갈피는 서재 담기라는 단일 규칙으로 정리했다.
- 서재 선택과 새 서재 생성은 iOS식 하단 sheet로 통일했다.
- 내 서재 화면의 새 서재 입력창은 기본으로 접어두고, 필요할 때 `+` 버튼으로 열리게 바꿨다.
- 비어 있는 서재에는 바로 검색으로 이동할 수 있는 짧은 액션을 추가했다.
- 내 서재의 선택 칩과 빈 상태 액션은 검은색 강조를 줄이고 시스템 회색 톤으로 낮췄다.
- 내 서재의 책 목록은 좌측 커버, 가운데 제목/저자/출판사, 우측 `빼기` 보조 액션 구조로 맞췄다.
- 내 서재 빈 상태와 서재 선택 영역의 시각 무게를 낮춰 검색 결과 카드와 같은 iOS식 리스트 톤으로 맞췄다.
- 서재 이름 변경, 공유, 비우기, 삭제 액션을 책 목록 헤더에서 분리하고 상단 서재 관리 메뉴 안으로 정리했다.
- 검색 결과와 상세 페이지에서 같은 책을 담으면 한 권으로 합쳐지도록 저장 키 기준을 제목/저자 중심으로 통일했다.
- 책 상세 상단 커버가 보이는 배경 영역 안에서 위아래 여백이 균형 있게 보이도록 위치를 조정했다.
- 검색 결과 카드나 홈 도서 카드에서 상세 페이지로 이동할 때 즉시 로딩 오버레이와 spinner를 보여주도록 했다.
- iPhone에서 내 서재의 새 서재 입력창을 열 때 화면이 확대되거나 가로 스크롤이 생기지 않도록 입력 필드 크기와 폼 폭을 조정했다.
- `상세`, `빼기`, `추가` 같은 텍스트 버튼은 줄이고, icon-only 버튼과 접근성 라벨로 정리했다.
- 저장된 책 카드에서는 불완전할 수 있는 보유 도서관 숫자를 강조하지 않고, 정확한 상태 확인은 상세 페이지에서 하도록 흐름을 맞췄다.

## 2026-05-12 - Soulib 모바일 검색 경험 개편

### 이번 버전에서 바뀐 점
이번 업데이트의 핵심은 Soulib을 "저장된 DB에서 검색 결과를 보여주는 서비스"에서 "사용자가 검색한 순간 실제 전자도서관 플랫폼을 직접 조회하는 서비스"에 가깝게 바꾼 것이다.

이전 방식은 미리 수집해 둔 DB를 기준으로 검색 결과를 구성했다. 그래서 화면은 빠르게 뜰 수 있지만, 실제 대출 가능 여부나 플랫폼별 최신 상태와 차이가 생길 수 있었다. 이번 버전부터 사용자 검색은 `교보`, `YES24`, `기타 도서관`을 직접 조회하는 흐름으로 전환했다. 사용자는 검색 시점에 더 가까운 결과를 보게 된다.

### 사용자 경험
- 홈 화면을 검색에만 집중하도록 중앙 정렬 검색 화면으로 정리했다.
- 홈에서도 하단 내비게이션을 유지해 다른 화면과 이동 구조를 맞췄다.
- 홈의 중앙 브랜드 워드마크를 제거하고, 서비스 제목과 검색바 중심으로 다시 정리했다.
- 정식 로고는 추후 제작 후 좌상단에 작은 브랜드 표시로 붙이는 방향으로 정했다.
- Soulib 아이콘은 검색바 안쪽에 배치했다.
- 홈 문구는 서비스 범위가 드러나도록 서울 공공 전자도서관, 교보, YES24 직접 검색 맥락을 보강했다.
- 검색창 아래에 있던 플랫폼 뱃지, 예시 검색 결과, 장식성 영역을 제거했다.
- 상단의 중복 검색 버튼을 제거하고, 하단 내비게이션 중심으로 이동 구조를 단순화했다.
- 하단 내비게이션은 `홈`, `검색`, `신고` 3개만 남겼다.
- 검색 결과의 공급사 분류는 `교보`, `YES24`, `기타`로 통일했다.
- `북큐브`는 별도 탭으로 빼지 않고 `기타`에 포함했다.
- 필터의 공급사 분류도 결과 카드와 같은 `교보`, `YES24`, `기타` 기준으로 맞췄다.
- 검색 탭 첫 진입 시 빈 화면 대신 짧은 검색 유도 상태를 보여주도록 정리했다.
- 화면 중간의 불필요한 회색 구분선과 넓이가 제각각이던 레이아웃을 정리했다.

### 디자인
- 전체 화면을 모바일 앱처럼 보이도록 하나의 앱 셸 기준으로 맞췄다.
- 홈, 검색, 책 상세, 오류 신고 페이지의 최대 폭을 동일하게 통일했다.
- PC에서 봐도 페이지마다 폭이 다르게 보이지 않도록 공통 컨테이너 폭을 다시 잡았다.
- 책 상세 페이지는 모바일과 PC 모두 커버/배경, 제목, 저자, 출판사 순서의 동일한 흐름을 유지하도록 정리했다.
- 색 사용을 줄이고, iPhone 기본 앱에 가까운 흰 배경, 얇은 경계, 차분한 회색 톤을 기준으로 정리했다.
- Pretendard 기반 타이포그래피를 유지한다.
- 신고 페이지의 분류 선택 화살표 위치를 정렬했다.
- iPhone 홈 화면에 추가했을 때 보이는 웹앱 아이콘을 투명 배경 없는 정사각형 아이콘으로 교체했다.

### 검색 및 상세
- 검색 API는 기존 DB 검색 중심에서 실시간 직접 조회 중심으로 바뀌었다.
- 검색 결과는 플랫폼별로 흩어져 보이지 않도록 같은 책으로 보이는 항목을 묶어 보여준다.
- 검색 결과 카드에서는 불완전하게 보일 수 있는 공급사/도서관 수량 뱃지를 제거하고, 카드 전체를 눌러 상세로 들어가는 리스트 셀 구조로 바꿨다.
- 책 상세 페이지에서는 모든 도서관 조회가 끝날 때까지 기다렸다가 한 번에 보여주는 느낌을 줄였다.
- 상세 페이지 카드가 먼저 보이고, 각 도서관의 상태가 확인되는 대로 순차적으로 갱신되도록 개선했다.
- 검색 결과에서 상세 페이지로 이동할 때 같은 실시간 검색을 다시 실행하지 않도록 캐시된 책 데이터를 사용한다.
- 제목/저자 통합 검색에서 제목 결과가 이미 확인된 플랫폼은 불필요한 추가 검색 요청을 줄였다.

### 오류 신고
- 사용자가 오류를 간단히 남길 수 있는 `/reports` 페이지를 추가했다.
- 신고할 수 있는 내용은 검색 결과 오류, 대출 상태 오류, 화면 오류 등이다.
- 신고를 보내면 방금 접수된 항목이 즉시 보이고, 최근 접수 목록 맨 위에도 바로 반영된다.
- 연락처는 선택 입력이며, 공개 목록에는 노출하지 않는다.

### 운영 메모
- Cloudtype 운영 앱은 `web/app_cloudtype.py`에서 `web/app_search.py`를 불러오는 구조다.
- 실시간 조회 기능은 `web/live_search/` 아래 커넥터와 정규화 로직을 사용한다.
- 실시간 검색은 전체 응답 대기를 줄이기 위해 도서관별 timeout과 전체 timeout을 둔다.
- 오류 신고는 GitHub Issues를 단일 저장소로 사용한다. DB/JSONL 파일 fallback은 사용하지 않는다.
- UX/UI 기준은 `docs/UX_UI_Guide.md`에 정리했다.
- GitHub `main` push 시 smoke test 후 공식 Cloudtype deploy action으로 자동 배포하는 workflow를 사용한다.
