# 릴리즈 노트

## 2026-05-12 - Soulib 모바일 검색 경험 개편

### 이번 버전에서 바뀐 점
이번 업데이트의 핵심은 Soulib을 "저장된 DB에서 검색 결과를 보여주는 서비스"에서 "사용자가 검색한 순간 실제 전자도서관 플랫폼을 직접 조회하는 서비스"에 가깝게 바꾼 것이다.

이전 방식은 미리 수집해 둔 DB를 기준으로 검색 결과를 구성했다. 그래서 화면은 빠르게 뜰 수 있지만, 실제 대출 가능 여부나 플랫폼별 최신 상태와 차이가 생길 수 있었다. 이번 버전부터 사용자 검색은 `교보`, `YES24`, `기타 도서관`을 직접 조회하는 흐름으로 전환했다. 사용자는 검색 시점에 더 가까운 결과를 보게 된다.

### 사용자 경험
- 홈 화면을 검색에만 집중하도록 정리했다.
- 검색창 아래에 있던 플랫폼 뱃지, 예시 검색 결과, 장식성 영역을 제거했다.
- 상단의 중복 검색 버튼을 제거하고, 하단 내비게이션 중심으로 이동 구조를 단순화했다.
- 하단 내비게이션은 `홈`, `검색`, `신고` 3개만 남겼다.
- 검색 결과의 공급사 분류는 `교보`, `YES24`, `기타`로 통일했다.
- `북큐브`는 별도 탭으로 빼지 않고 `기타`에 포함했다.
- 화면 중간의 불필요한 회색 구분선과 넓이가 제각각이던 레이아웃을 정리했다.

### 디자인
- 전체 화면을 모바일 앱처럼 보이도록 하나의 앱 셸 기준으로 맞췄다.
- 홈, 검색, 책 상세, 오류 신고 페이지의 최대 폭을 동일하게 통일했다.
- PC에서 봐도 페이지마다 폭이 다르게 보이지 않도록 검색 결과도 모바일형 단일 열로 유지한다.
- 색 사용을 줄이고, iPhone 기본 앱에 가까운 흰 배경, 얇은 경계, 차분한 회색 톤을 기준으로 정리했다.
- Pretendard 기반 타이포그래피를 유지한다.
- 신고 페이지의 분류 선택 화살표 위치를 정렬했다.

### 검색 및 상세
- 검색 API는 기존 DB 검색 중심에서 실시간 직접 조회 중심으로 바뀌었다.
- 검색 결과는 플랫폼별로 흩어져 보이지 않도록 같은 책으로 보이는 항목을 묶어 보여준다.
- 책 상세 페이지에서는 모든 도서관 조회가 끝날 때까지 기다렸다가 한 번에 보여주는 느낌을 줄였다.
- 상세 페이지 카드가 먼저 보이고, 각 도서관의 상태가 확인되는 대로 순차적으로 갱신되도록 개선했다.

### 오류 신고
- 사용자가 오류를 간단히 남길 수 있는 `/reports` 페이지를 추가했다.
- 신고할 수 있는 내용은 검색 결과 오류, 대출 상태 오류, 화면 오류 등이다.
- 최근 접수된 신고를 간단히 확인할 수 있다.
- 연락처는 선택 입력이며, 공개 목록에는 노출하지 않는다.

### 운영 메모
- Cloudtype 운영 앱은 `web/app_cloudtype.py`에서 `web/app_search.py`를 불러오는 구조다.
- 실시간 조회 기능은 `web/live_search/` 아래 커넥터와 정규화 로직을 사용한다.
- 오류 신고는 운영 DB의 `error_reports` 테이블에 저장한다. DB 연결이 없을 때는 로컬 JSONL 파일로 임시 저장한다.
- UX/UI 기준은 `docs/UX_UI_Guide.md`에 정리했다.
