블로그 글 추가 방법

1. 이 폴더에 `영문-slug.md` 파일을 추가합니다.
2. 아래 형식을 복사해서 작성합니다.
3. 저장 후 `/blog`에서 자동으로 보이는지 확인합니다.

현재 공개 카테고리는 두 개입니다.

- `이용 안내` (`category_slug: guide`): Soulib 사용법, 서울온 준비, 전자책 앱 설치, 대출 상태, 오류 신고 등 이용 흐름 안내
- `책 추천` (`category_slug: recommendations`): 신간, 베스트셀러, 주제별 추천

예시:

---
title: 글 제목
description: 목록에 보이는 짧은 설명
category: 이용 안내
category_slug: guide
date: 2026-05-21
---

## 소제목

본문을 짧게 씁니다.

- 목록 항목 1
- 목록 항목 2

![이미지 설명](/static/img/blog/example.png)

지원하는 문법은 `##`, `###`, 문단, `-` 목록, 외부 링크, `/static/` 이미지, `**굵게**`, `==형광펜==` 정도입니다.

책 추천 글에서는 아래 Soulib 검색 카드 문법을 사용할 수 있습니다. 제목, 저자/구분, 추천 이유, 실제 검색어 순서입니다. 검색어를 생략하면 제목과 저자/구분을 합쳐 검색합니다. 단, 본문에서는 이 블록을 설명하지 않습니다.

```text
[[soulib-search:마션|앤디 위어|문제 해결형 SF를 바로 찾고 싶을 때|마션 앤디 위어]]
```

목차 링크가 필요하면 제목 뒤에 `{#영문-id}`를 붙입니다.

## 검색하기 {#guide-search}

편집 기준:

- 처음 이용자가 실제로 따라 할 수 있는 순서로 씁니다.
- 핵심 행동은 `**굵게**`, 놓치면 안 되는 주의사항은 `==형광펜==`으로 표시합니다. 한 문단에 강조를 너무 많이 넣지 않습니다.
- Soulib이 해주는 일과 각 도서관, 공급사, 공식 앱에서 해야 하는 일을 구분합니다.
- 앱 설치 링크, 회원증 안내, 도서관 정책은 공식 사이트나 공식 앱스토어 링크만 사용합니다.
- 서울온은 서울 도서관 회원증과 시설카드 관리를 시작하는 추천 출발점으로 설명하되, 모든 도서관 전자책 이용이 자동으로 열리는 것처럼 쓰지 않습니다.
- 화면 설명이 필요한 글은 실제 스크린샷이나 설명 이미지를 함께 준비합니다. 임시 이미지나 공개 TODO 문구는 본문에 넣지 않습니다.

우선 개선할 글:

- `seoul-on-library-guide.md`: 서울온 설치, 가입, 시설카드 추가, 도서관별 인증 caveat, 앱 설치 순서를 가장 먼저 보강합니다.
- `soulib-guide.md`: 이미 보유한 Soulib 화면 스크린샷을 기준으로 검색, 상세, 내 서재, 오류 신고 흐름을 유지 보강합니다.
- `ebook-library-status-guide.md`: 대출 상태 용어를 실제 행동으로 연결하고, 상세 화면 상태 예시 이미지를 추가하면 좋습니다.

자동 작성 품질 기준:

- 자동화는 매 실행마다 최대 1개 글만 작성합니다. 품질 기준을 못 맞추면 새 글을 만들지 않습니다.
- 자동 발행 글은 `python scripts/blog_quality_check.py --strict content/blog/<slug>.md`를 통과해야 합니다.
- 주제는 기존 글과 중복되지 않아야 합니다. 같은 결론을 제목만 바꿔 반복하는 글은 발행하지 않습니다.
- 기본 분량은 본문 3,500자 이상입니다. 단, 분량보다 실제 문제 해결 순서와 검증된 정보가 우선입니다.
- 글의 첫 부분에서 누구의 어떤 문제를 해결하는지와 추천 행동 순서를 바로 말합니다.
- 도서관 정책, 회원 자격, 앱 설치, 대출 조건은 공식 출처로 확인한 내용만 단정합니다.
- Soulib이 직접 대출, 예약, 반납을 처리한다고 오해하게 쓰면 안 됩니다.
- 화면 설명이 필요한 주제는 실제 `/static/img/blog/...` 이미지가 준비된 경우에만 발행합니다.
- 책 추천 글은 최소 3개 이상의 Soulib 검색 카드 블록을 넣되, 본문 문장에서는 `검색 카드`라고 부르지 않습니다.
- 책 추천 글의 Soulib 검색 카드 블록은 상단에 몰아두지 말고, 해당 책을 설명하는 섹션 바로 위나 아래에 둡니다.
- 책 추천 글의 대표 이미지는 관련 책의 실제 표지 이미지 세트나 해당 책 대표 이미지를 로컬 정적 파일로 저장해 사용합니다. 기본 경로는 `/static/img/blog/recommendations/<topic>/hero-cover-set.png`입니다.
- 외부 이미지 핫링크는 금지합니다. 표지 이미지는 `web/static/img/blog/book-covers/` 또는 관련 추천글 폴더 아래 로컬 파일로 저장하고, 출처는 `web/static/img/blog/book-covers/SOURCES.md`에 기록합니다.
- 특정 책 화면 스크린샷은 그 책을 직접 다루는 글에만 씁니다. 다른 주제의 추천 글에 프로젝트 헤일메리 같은 무관한 검색 결과 이미지를 재사용하지 않습니다.
- 추천 도서마다 본문은 책의 성격, 독자 상황, 함께 읽을 이유를 중심으로 씁니다. Soulib 검색은 카드 블록으로만 연결하고, 별도 지시문처럼 반복하지 않습니다.
- 추천 도서 섹션 제목은 책 제목이나 자연스러운 읽기 맥락으로 씁니다. `후보 1`, `추천 1`, `첫 번째 후보`처럼 번호표가 앞에 붙은 제목은 쓰지 않습니다.
- 추천글 본문에는 `정확한 Soulib 검색어`, `Soulib 검색어`, `결과에서 확인할 것`, `제목+저자`, `제목 전체와 저자명`, `검색 카드`, `오늘 바로 실행`, `행동으로 바꾸는` 같은 유도 문구를 쓰지 않습니다.
- 표지, 출판사, 저자명을 대조하라는 지시를 책마다 반복하지 않습니다. 필요한 경우 글 전체에서 한 번만 자연스럽게 안내합니다.
- 외부 글, 기사, 블로그, 공식 안내를 실제로 참조했다면 하단 출처 목록을 만들지 말고 본문 안의 관련 단어 또는 짧은 문장에 바로 링크를 겁니다. 예: `전자책 이용 기준은 [서울도서관 안내](https://...)에서 확인할 수 있습니다.`
- 참조 링크는 독자가 더 확인하고 싶은 주장, 통계, 정책, 기사, 공식 안내에만 붙입니다. 참조하지 않은 링크를 신뢰도 장식처럼 붙이지 않습니다.
- 지원하지 않는 Markdown 문법에 핵심 정보를 넣지 않습니다. 표, 상대 내부 링크, HTML 조각에 의존하지 않습니다.
- SEO 키워드는 자연스럽게만 사용합니다. 검색어 나열, 일반론 반복, 광고 문구는 저품질로 봅니다.

발행 전 검증:

- frontmatter의 `title`, `description`, `category`, `category_slug`, `date`가 모두 있어야 합니다.
- `category_slug`는 `guide` 또는 `recommendations`만 사용합니다.
- 이미지가 있으면 실제 파일이 존재해야 하고, `image_alt`와 본문 alt/caption이 구체적이어야 합니다.
- 내부 링크가 필요하면 `/blog/<slug>` 상대경로 대신 `https://www.soulib.kr/blog/<slug>` 형식을 씁니다.
- 자동 발행 글은 `##` 섹션 5개 이상, 구체적 행동을 표시한 `**굵게**`, 중요한 오해 방지를 표시한 `==형광펜==`을 포함해야 합니다.
- 이용 안내 글은 공식 `https://` 출처 링크를 2개 이상 포함해야 합니다.
- 하단 `## 참고한 자료`, `## 출처`, `## References` 섹션은 만들지 않습니다. 필요한 reference는 본문 흐름 안에서 인라인 링크로 처리합니다.
- `TODO`, 표 문법, 번호 목록, 원시 HTML, 존재하지 않는 이미지, 지원하지 않는 상대 링크가 있으면 발행하지 않습니다.
- `python scripts/smoke_test.py` 또는 동등한 블로그 렌더링 검증 없이 배포하지 않습니다.

자동 작성 에이전트 발행 순서:

- 먼저 `content/blog/`의 기존 글 제목, 결론, 섹션 흐름을 요약하고 중복 주제를 제외합니다.
- 새 글의 독자 문제, 검색 의도, 공식 출처 URL, 섹션 설계를 먼저 정합니다.
- 초안을 작성한 뒤 스스로 저품질 판정 기준을 적용해 하나라도 걸리면 파일을 삭제하고 발행하지 않습니다.
- 발행 후보 파일에 `python scripts/blog_quality_check.py --strict content/blog/<slug>.md`를 실행합니다.
- 이어서 `python scripts/smoke_test.py`와 `git diff --check`를 실행합니다.
- 위 검증 중 하나라도 실패하면 커밋하지 않습니다.

우선 주제 큐:

- 교보 전자도서관 앱에 대출한 책이 안 뜰 때
- YES24 전자도서관 로그인과 기관 선택 체크리스트
- 북큐브, 부커스, 구독형 전자책 차이
- 전자책 예약 순번, 반납 예정일, 예약 만료일 보는 법
- 서울 구립도서관 전자책 이용 자격 확인법
- 전자도서관 검색 결과가 너무 많을 때 좁히는 법
- 아이와 읽을 전자책을 Soulib에서 찾는 법
- 전자도서관에서 읽기 좋은 SF, 추리, 경제경영 책 찾는 법
